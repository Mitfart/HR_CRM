1. Общее описание системы
CRM-платформа автоматизирует полный цикл работы кадрового агентства: от первичной заявки клиента до подписания договора. Система включает мультиканальных ботов, встроенный календарь собеседований, интеграцию с видеосервисами (Толк, Яндекс Телемост, Мост) и автоматическую генерацию юридических документов.

1.1 Цели проекта
Автоматизировать сбор заявок и первичное вовлечение клиентов через ботов
Обеспечить согласование времени собеседования через встроенный календарь CRM
Интегрировать платформу с видеосервисами: Толк, Яндекс Телемост, Мост
Автоматически генерировать договор после успешного собеседования
Ускорить подбор соискателей из базы данных по параметрам клиента
Хранить все данные в зашифрованном виде согласно нормам законодательства РФ

1.2 Роли пользователей
Роль
Описание и права доступа
Клиент
Заполняет форму, общается с ботом, согласовывает время собеседования, получает и подписывает договор
Менеджер
Работает в CRM: ведёт заявки, управляет календарём, инициирует поиск соискателей и формирует договоры
Соискатель
Находится в базе; получает предложения от агентства через мессенджеры
Администратор
Настраивает скрипты бота, шаблоны договоров, интеграции, права доступа

3. Модули системы
3.1 Публичный сайт — лендинг + форма заявки
Описание услуг агентства (контент управляется из CMS-раздела в админке)
Форма заявки: описание потребности, контакты: Telegram, WhatsApp, MAX, Email
После отправки: экран подтверждения «Мы уже пишем вам!»
SEO-оптимизация, адаптивная вёрстка для мобильных устройств

3.2 Мультиканальный бот (Bot Engine)
Канал
Технология / API
Telegram
Telegram Bot API (aiogram 3.x, Python)
WhatsApp
Green API (неофициальная, быстрый старт)
MAX (Mail.ru)
MAX Bot API (VK/Mail.ru платформа)
Email
SMTP / SendGrid / Mailgun

Логика бота:
Приветствие + краткое описание агентства
Уточняющие вопросы по заготовленному скрипту (возраст, требования, режим)
Отправка ссылок на внешние сервисы в зависимости от типа заявки
Предложение выбрать время собеседования — слоты из календаря CRM
Подтверждение записи + напоминание за 1 час до созвона
Ответы на типичные вопросы клиента (FAQ-блок)

Скрипты вопросов, FAQ и ссылки на сервисы настраиваются администратором в визуальном редакторе CRM без изменения кода.

3.3 Интеграция с видеосервисами (Толк / Яндекс / Мост)
После подтверждения времени собеседования система автоматически:
Выбирает нужный сервис в зависимости от типа заявки или предпочтения клиента
Создаёт ссылку на видеовстречу и отправляет её клиенту и менеджеру
Добавляет событие в календарь CRM с прямой ссылкой на звонок

Сервис
Описание интеграции
Толк (VK Calls)
Корпоративный видеосервис от VK. Подходит если клиент использует VK/MAX. Создание комнаты через API.
Яндекс Телемост
Видеосервис Яндекса. Широко распространён в РФ. Создание ссылки через Яндекс API / OAuth.
Мост
Отечественный сервис видеоконференций. Интеграция через REST API. Дополнительная опция.

3.4 CRM-панель для менеджеров
3.4.1 Доска заявок (Kanban)
Колонки: Новая → Бот опросил → Собеседование назначено → Собеседование прошло → Поиск соискателей → Договор отправлен → Закрыта
Карточка: имя клиента, контакты, статус бота, время собеседования, ссылка на видеосвязь
Автоматическое обновление данных в реальном времени (WebSocket)

3.4.2 Встроенный календарь
Визуальный календарь с недельным и месячным видом
Менеджер задаёт свободные слоты — бот предлагает их клиенту
После подтверждения клиентом слот блокируется и создаётся событие с ссылкой на видеосервис
Напоминания: за 24 часа и за 1 час до встречи — клиенту и менеджеру
Синхронизация с Google Calendar / Яндекс Календарь (опционально)

3.4.3 Карточка заявки
История переписки бота с клиентом
Кликабельные контакты (открыть в Telegram, WhatsApp и т.д.)
Поле для заметок менеджера после собеседования
Параметры поиска соискателя — заполняются после созвона
Кнопки: "Найти соискателей", "Создать договор"

3.4.4 Поиск и выбор соискателей
Поиск по базе: специализация, возраст, зарплата, опыт, доступность
Менеджер выбирает соискателей → отправляет им предложение через бота
Статус каждого: Отправлено / Согласен / Отказал
Выбор финальных кандидатов → отправка клиенту

3.5 Модуль генерации договоров
После согласования кандидата система автоматически формирует договор:
Шаблоны договоров создаются администратором в редакторе (переменные: ФИО, услуги, стоимость, сроки)
Данные подставляются автоматически из карточки заявки и профиля соискателя
Готовый договор отправляется клиенту через бота (PDF) или на email
Договор соответствует нормам законодательства РФ (ГК РФ, ТК РФ)
Все версии договоров хранятся в CRM с историей изменений

На первом этапе MVP — генерация PDF-договора из шаблона с авто-подстановкой данных. Электронная подпись (КЭП) — отдельный этап после запуска.

3.6 Безопасность и шифрование
Все персональные данные клиентов шифруются в базе данных (AES-256)
Передача данных только через HTTPS (TLS 1.3)
JWT-авторизация для сотрудников, 2 роли: admin / manager
Логирование всех действий менеджеров (аудит-лог)
Хранение данных на серверах в РФ (соответствие 152-ФЗ «О персональных данных»)

3.7 База данных соискателей
Карточка соискателя: ФИО, возраст, специализация, опыт, зарплата, доступность, контакты
Импорт из текущей CRM-системы (CSV/JSON — модуль миграции)
Добавление вручную через форму в CRM
История предложений: какие заявки отправлялись, какие приняты

3.8 Модуль уведомлений
Менеджеру: новая заявка, ответ клиента боту, подтверждение времени, ответ соискателя
Клиенту: подтверждение записи на собеседование, напоминание за 1 час, договор готов
Соискателю: предложение о работе через их канал связи
Каналы: Push в браузере, Email, Telegram-уведомления менеджеру
