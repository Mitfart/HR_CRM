## 3. Роли пользователей

### Администратор (`admin`)

- Полный доступ ко всему
- Управляет пользователями (создаёт менеджеров и других админов)
- Редактирует скрипты бота
- Видит всё что видит менеджер

**Страницы:** `/crm`, `/crm/admin`

---

### Менеджер (`manager`)

- Работает с канбан-доской заявок
- Открывает карточки заявок, читает переписку с ботом
- Добавляет заметки к заявкам
- Ищет соискателей и отправляет им предложения
- Видит ответы соискателей (принял / отказал) 
- Работает с базой соискателей

**Страницы:** `/crm`, `/crm/candidates`, `/crm/applications/[id]`

---

### Клиент (`client`)

- Видит только свои заявки и их статус
- Не видит других клиентов

**Страница:** `/client`

---

### Соискатель (`candidate`)

- Создаёт и редактирует своё резюме / профиль

**Страница:** `/candidate`

---

## 4. Как работает система — полный цикл

```
1. Клиент заходит на сайт (/) и заполняет форму заявки
   → указывает Telegram / WhatsApp / email / описание потребности

2. Менеджер получает новую заявку в CRM
   → уточняющие вопросы и коммуникация ведутся вручную или через будущие подключённые каналы

3. Менеджер видит новую заявку в CRM (/crm)
   → карточка появляется в колонке "Новая"
   → история сообщений сохраняется в карточке заявки

4. Менеджер звонит клиенту, уточняет требования
   → меняет статус карточки вручную (drag & drop или select)
   → добавляет заметки после звонка

5. Менеджер нажимает "Найти соискателей"
   → система ищет по базе (опыт, зарплата, специализация)

6. Менеджер выбирает подходящих кандидатов
   → предложение фиксируется в CRM, отправка через Telegram-бот отключена

7. Соискатели отвечают (согласен / отказал)
   → менеджер обновляет статусы в карточке заявки

8. Менеджер отправляет финальных кандидатов клиенту
   → клиент видит их в своём личном кабинете (/client)

9. Заявка закрывается
```

## 5. CRM — Доска заявок (`/crm`)

### Колонки канбан


| Колонка            | Значение               |
| ------------------ | ---------------------- |
| Новая              | только что поступила   |
| В работе           | менеджер взял в работу |
| Ожидает звонка     | договорились о созвоне |
| Поиск соискателей  | ищем кандидатов        |
| Отправлено клиенту | кандидаты предложены   |
| Собес              | идут видеоинтервью     |
| Закрыта            | сделка завершена       |


### Что можно делать

- Перетаскивать карточки между колонками (drag & drop)
- Менять статус через выпадающий список прямо на карточке
- Фильтровать по каналу связи (Telegram / WhatsApp / MAX / Email)
- Искать по тексту описания и контактам
- Открыть боковую панель с деталями — кликнуть на карточку
- Открыть полную страницу заявки — ссылка "Открыть отдельную карточку"
- Зафиксировать исходящее сообщение клиенту в CRM

## 6. CRM — Настройки (`/crm/admin`) — только для admin

> Кнопка "Настройки" появляется в шапке CRM только у администратора.

### Вкладка «Пользователи»

- Список всех пользователей системы с ролями и статусом
- **«+ Добавить пользователя»** — создать менеджера или нового админа
  - Заполнить: Email, Полное имя, Пароль, Роль
- **«Отключить»** / **«Активировать»** — заблокировать доступ без удаления

### Вкладка «Скрипты бота»

- Список вопросов которые бот задаёт клиенту по очереди
- **«+ Добавить вопрос»** — добавить новый вопрос в конец
- Редактировать текст — прямо в поле
- Переставить порядок — кнопки ▲ ▼
- Удалить вопрос — кнопка ✕
- **«Сохранить»** — сохранить все изменения в базе данных

> Изменения применяются сразу ко всем новым диалогам бота.

## 8. Сервер — управление

### Подключиться по SSH

```bash
ssh root@62.181.53.36
# пароль хранится вне репозитория
```

### Основные команды

```bash
# Перейти в папку проекта
cd /root/CRM

# Статус всех контейнеров
docker compose ps

# Логи конкретного сервиса (live)
docker compose logs backend --tail=50 -f
docker compose logs frontend --tail=50 -f
docker compose logs nginx --tail=50 -f

# Перезапустить один сервис
docker compose restart backend

# Перезапустить всё
docker compose restart

# Остановить всё
docker compose down

# Запустить всё
docker compose up -d
```

### Обновление проекта после изменений в коде

```bash
cd /root/CRM

# Пересобрать только изменившийся сервис (например frontend)
docker compose build frontend
docker compose up -d --force-recreate frontend

# Или пересобрать всё
docker compose build
docker compose up -d

# Применить новые миграции БД (если были изменения в моделях)
docker compose exec backend alembic upgrade head
```

### Резервная копия базы данных

```bash
# Создать бэкап
docker compose exec postgres pg_dump -U crm_user crm_agency > backup_$(date +%Y%m%d).sql

# Восстановить из бэкапа
docker compose exec -T postgres psql -U crm_user crm_agency < backup_20260410.sql
```

## 9. Файл настроек `.env`

Находится на сервере: `/root/CRM/.env`

```bash
# Посмотреть
cat /root/CRM/.env

# Отредактировать
nano /root/CRM/.env

# После изменений перезапустить нужный сервис
docker compose restart backend
```

### Ключевые переменные


| Переменная                                  | Назначение                              |
| ------------------------------------------- | --------------------------------------- |
| `SECRET_KEY`                                | Ключ подписи JWT токенов                |
| `DATABASE_URL`                              | Строка подключения к PostgreSQL         |
| `REDIS_URL`                                 | Строка подключения к Redis              |
| `GREEN_API_INSTANCE` / `GREEN_API_TOKEN`    | WhatsApp через Green API                |
| `MAX_BOT_TOKEN`                             | Токен бота MAX (Mail.ru)                |
| `SMTP_HOST` / `SMTP_USER` / `SMTP_PASSWORD` | Email уведомления                       |

## 11. Что ещё не настроено


| Что                   | Что нужно сделать                                                                              |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| **HTTPS / SSL**       | Купить домен, привязать к IP, настроить Let's Encrypt                                          |
| **WhatsApp**          | Зарегистрироваться на green-api.com, вписать `GREEN_API_INSTANCE` и `GREEN_API_TOKEN` в `.env` |
| **MAX (Mail.ru)**     | Создать бота в экосистеме MAX, вписать `MAX_BOT_TOKEN` в `.env`                                |
| **Email уведомления** | Настроить SMTP (например Gmail App Password) в `.env`                                          |
| **Домен**             | Купить домен и прописать A-запись на `62.181.53.36`                                            |


### Как подключить SSL после покупки домена

```bash
# На сервере установить certbot
apt install certbot python3-certbot-nginx -y

# Остановить наш nginx контейнер
cd /root/CRM && docker compose stop nginx

# Получить сертификат
certbot certonly --standalone -d ваш-домен.ru

# Скопировать сертификаты в проект
cp /etc/letsencrypt/live/ваш-домен.ru/fullchain.pem /root/CRM/nginx/certs/
cp /etc/letsencrypt/live/ваш-домен.ru/privkey.pem /root/CRM/nginx/certs/

# Раскомментировать HTTPS блок в nginx/nginx.conf
# Запустить nginx обратно
docker compose up -d nginx
```
